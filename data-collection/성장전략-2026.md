# 아기머니 성장전략 최종 리포트 (부산 1인·코딩초보용)

## 1. 핵심 진단 (한 줄)

**리텐션 1%는 "고쳐야 할 결함"이 아니라 "지원금은 생애 몇 번만 조회"라는 카테고리 특성이다. 그러니 매일 여는 습관앱을 억지로 만들지 말고, ① 지금 새는 첫 세션 광고 노출(164명→28명, 17%)을 막아 볼륨을 즉시 3~5배 올리고, ② 개월수·마감일이라는 "시간이 스스로 만들어주는 재접점"으로 몇 달에 걸친 재방문을 설계하며, ③ 그 모든 것을 떠받치는 "부산 검증 데이터 엔진" 하나에 집중 투자하라.** eCPM 86,000원(단가 최상)인데 노출이 28회뿐 — 병목은 콘텐츠나 단가가 아니라 노출 볼륨이다.

---

## 2. 🔁 리텐션 실행안 (우선순위·시퀀스)

리텐션은 "정기(동네 소식) + 비정기(내 아이 마일스톤)" **2트랙 푸시 엔진 하나로 통합**한다. 개별 기능을 5개 따로 만들지 말 것.

### [R1] 첫 세션 광고 퍼널 계측·복구 — 가장 먼저 (임팩트 高 / 노력 低)
- 리텐션 기능에 손대기 전, 조회 완료 164명 중 광고 노출이 28명(17%)뿐인 원인을 로그로 특정한다. `조회완료→결과요청→광고요청→광고로드→노출완료` 각 단계에 이벤트 로깅을 붙여 136명이 새는 지점 **하나**를 찾는다(30분~며칠).
- 광고 "강제"는 토스 리워드 정책(자발적 시청) 위반 소지 → 대신 **결과 요약 먼저 노출 → 상세(지자체별 금액)는 광고 후**로 게이트를 이탈 지점 뒤로 미룬다.
- 프레이밍 주의: 이건 "수익 안"이 아니라 **이후 모든 판단을 위한 데이터 정상화 안**이다. 절대금액이 월 몇천원이라 노출 3~5배로도 아직 큰돈은 아니고, eCPM 86,000원은 노출 28회 극소표본이라 그대로 곱하면 안 된다.

### [R2] 개월수 생애주기 마감 알림 (2트랙 중 "비정기·내 아이") — 최우선 구조적 레버 (임팩트 高 / 노력 中)
- 이미 입력받는 생년월일 → 개월수는 앱이 가만있어도 시간이 "다시 열 이유"를 만든다. 5개 중 **유일한 구조적 리텐션 레버**.
- 12~24개월 전체 타임라인을 미리 심지 말고, **데이터가 확실한 3~4개 고가치 트리거에만 집중**: 출생 60일 신청 마감, 첫돌축하금 D-14, 어린이집 입소창, 특례대출 자녀요건. (연 4회는 과장, 실제 연 2~3회지만 0→2~3회는 압도적 개선)
- 구현: 개인별 예약 푸시(토스 스마트발송 미지원 가능성) 대신 **매주 크론 배치로 "이번 주 트리거에 걸린 유저" 세그먼트 발송**. 초보 바이브코딩 범위 내.
- 푸시 문구엔 **손실회피 카피 편입**: "OO구 첫돌축하금 아직 신청 안 하셨어요(마감 D-N)", "OO구 기준 아직 안 받은 지원금 총 OOO만원".

### [R3] 격주/월간 "우리 동네 다이제스트" (2트랙 중 "정기·동네") (임팩트 中 / 노력 中)
- 원래의 "주간" 다이제스트와 "지역 변동 알림"을 여기로 **흡수·통합**. 주간은 버린다 — 지자체 데이터 변동 속도(연/분기 단위)와 정면 충돌해 소재 고갈→대출정보로 때움→개인화 붕괴→알림 차단의 역효과.
- 실제 데이터 속도에 맞춰 **격주/월간 고정 카덴스**로 구독 지역 변동분을 모아 발송. 소재 없는 주에 억지 발송하지 않는 것이 신뢰 유지에 낫다.
- 구독(구/동 저장) 로직은 지금 깔아둔다 — 향후 SaaS 자산.

### 버리거나 축소 (revise 반영)
- **체크리스트 게임화 → 폐기.** 실제 신청은 정부24/구청에서 끝나 "완료 체크하러 돌아올" 동기 없음. 유효 성분인 "미수령 총액 카피"만 뽑아 R2 푸시로 편입.
- **카톡 공유 프리필 링크 → 착수 보류·최소 투자.** 전체가 "토스 미니앱 지역 프리필 딥링크 지원" 가정에 걸려 있음. **먼저 30분 안에 토스 딥링크/공유 지원 여부 확인.** 미지원이면 "결과 카드 이미지 + 외부 웹 랜딩" 폴백. 지금은 "공유 버튼 + 클릭 계측"만 깔아 가설만 검증.

## 3. 💰 수익화 실행안 (광고 → 고단가 제휴, 유닛이코노믹스)

**볼륨이 구조적으로 안 나오는 앱의 정답은 "노출 늘리기"가 아니라 "1인당 전환단가 극대화" + "볼륨 무관 로컬 영업".** 유닛 비교: 리워드 광고 1회 = 86원 vs 보험 리드 1건 = 1~3만원 = 광고 200~350회분.

### [M1] 마감·신규 지원금 알림은 전부 "무료" — 광고 노출 배수화 (임팩트 高 / 노력 中)
- **자기모순 교정: 리텐션을 살릴 유일한 기능(마감 알림)을 2,900원 페이월 뒤에 잠그면 볼륨이 안 늘어 광고수익도 안 는다.** R2 알림을 전부 무료로 풀어 재방문→광고 노출 증가로 수익화. 결제/페이월은 나중.
- 유닛: eCPM 86,000원 유지 시 노출만 배수로 늘면 매출도 배수. (오늘 2,416원)

### [M2] 태아·어린이 보험 리드 (GA 제휴 CPA) — 아웃링크로 부담 0 시작 (임팩트 高 / 노력 中)
- 직접 폼 대신 **GA가 이미 운영하는 심의필 랜딩(동의 UI 완비)으로 아웃링크하는 순수 제휴배너**로 시작 → 법적·개발 부담 0. "송객만 하니 모집자격 불필요"는 낙관적 해석 → 보험업법상 모집 유사행위 소지, 반드시 GA 승인문구를 **계약서로 담보**.
- **유닛이코노믹스 재계산 필수**: 전환율은 제안의 5~10%가 아니라 **1~2%가 현실 상한**(D1 1%·신뢰 낮은 앱). 164명×1~2% = 월 2~4건. 계약 전 GA에 "월 10건 미만도 유지하나" "유효 리드 인정 기준(유효 통화)"을 먼저 확인.
- 스팸 인식(리텐션 추가 훼손) 방지: "전화 대신 카톡 상담" 옵트인.

### [M3] 부산·경남 산후조리원 — "리드 계약" 아닌 "수동 컨시어지"로 검증 (임팩트 高 / 노력 高)
- **가장 날카로운 통찰: 1인 코딩초보에게 무기는 코드가 아니라 부산 로컬 영업.** 조리원 LTV 300~500만원. 단, 현재 볼륨(월 164명 중 부산·경남 임신 임박 = 한 자릿수)으로 리드 계약을 맺으면 조리원이 몇 달 안에 이탈.
- **앱에 리드폼·자동화 붙이기 전에**, 조리원 1곳과 "성사 시 성공보수"만 구두 합의 → 부산 유저를 카톡/전화로 **수동 연결하는 컨시어지로 3개월 검증**. 실제 예약 1~2건이라도 나오면 그 실적으로 정식 계약. **월 10건 임계치 전엔 개발 투자 금지.**

### [M4] 신생아 특례대출 = 트래픽 미끼, 수익화는 안 함 (임팩트 中 / 노력 低)
- 금소법·대출모집인 등록(1사 전속) 문제로 **직접 CPA·리드·수수료 모델은 접는다.** 정책자금이라 애초에 수수료 붙일 구조도 아님. 지뢰 제거용 방어수.
- 대출 섹션은 자체 텍스트 대신 **정부/은행 공식 페이지 딥링크 + 기준일 자동 표기**로 최신성 부담을 외부로 넘김. 수익은 리워드 광고로만.

## 4. 🚀 SaaS 확장 로드맵 (데이터 엔진 축, 4축)

모든 확장의 전제이자 유일한 해자는 **"부산 지자체 지원금을 정확·검증해 모으는 데이터 엔진"** 하나. 이걸 여러 표면(웹/종이/버티컬/API)으로 재사용해 한계비용 0으로 노출·수익 표면을 늘린다.

### [축0] 데이터 엔진 — 자동=탐지, 수동=검증 (전제 인프라, 임팩트 高 / 노력 中으로 축소)
- 풀 자동 스크래핑 파이프라인은 **초보의 유지보수 함정**(PDF/한글파일/전화확인 많아 HTML diff 불가, 파서 깨짐, LLM 오류가 자산 훼손). 164명 규모에 과잉설계.
- 축소판: **부산 16개 구 구글시트 마스터 DB**를 단일 소스로 구축 → 앱/웹이 이 시트 하나만 읽게. 자동화는 "변경 감지"까지만(LLM이 주1회 구청 페이지 요약→"바뀐 것 같은 곳"만 플래그), **금액·조건 확정은 본인 수동**. 파서로 값을 직접 DB에 쓰지 말 것.
- **데이터를 표준 스키마로 저장**: `지역·지원항목·금액·신청기한·근거조례·기준일·출처URL`. 이 포맷 자체가 R2 마감 트리거 + 확장 + 라이선스의 씨앗.

### [축1 지역] 부산·동남권만 "압도적으로 정확하게" (임팩트 高 / 노력 中)
- 전국 226개 얕은 커버는 **유지보수 부채이자 신뢰 리스크**(핵심가치가 정확·최신인데 틀린 지자체 하나가 앱 전체 신뢰 붕괴 = 자멸). 부산 15개 구·군 + 경남 일부만 조례 원문·기준일까지 붙여 좁게 판다.
- 마케팅도 "부산·경남 출산지원금 가장 정확한 앱"으로 로컬 특화. 좁은 지역은 혼자 최신 유지 가능 + 맘카페·주민센터 구전 해자.

### [축2 플랫폼] SEO 롱테일 — 양산 아닌 파일럿 (임팩트 高 / 노력 中)
- "재방문 짜지 말고 매번 새로 검색하는 사람을 잡아라"는 정공법. 단 "부산 X구 출산지원금"은 정부24·맘카페가 선점한 레드오션(상위노출 3~6개월+).
- 수백 개 양산 대신 **부산 상위 3~5개 구만 "정부24보다 나은" 고품질 페이지로 파일럿** → 네이버/구글 키워드 도구로 검색 수요 먼저 확인 → 노출/순위 실측 후에만 확장.
- **토스 미니앱은 웹 색인이 안 됨** → 웹 페이지 자체를 수익 표면으로: 웹에서 바로 조회+웹 광고, 미니앱 CTA는 보조. 토스 진입 마찰을 병목에서 제거.

### [축3 오프라인 유통] 웹 위젯 아닌 "종이 한 장 요약" (임팩트 中 / 노력 中)
- iframe/script 위젯은 현실과 어긋남(네이버 카페는 임베드 불가, 동네 조리원·소아과 사이트 트래픽 0, 토스 리워드 진입 매끄럽지 않음).
- 대신 데이터 엔진에서 **"우리 동네(OO구) 출산지원금 한 장 요약" PDF 자동 생성** → 산후조리원·소아과·주민센터 비치용 무료 배포, 하단에 미니앱 QR + 갱신일. 종이→QR이 웹→앱보다 임신·출산 맥락에서 전환 마찰이 낮다.

### [축4 버티컬] 지금 확장 금지, 값싼 실험으로 수요만 확인 (임팩트 中 / 노력 高)
- "신혼→청년→부모 돌봄으로 가구 단위 재방문" 논리는 매력적이나 근거 약함: 근거인 "40대 최다"가 6/30 유입 이벤트 왜곡일 가능성, 재방문 주기가 몇 년 단위라 리텐션 인과 과장, 유지보수 부담 곱셈.
- **부산 출산·특례대출을 "정부24보다 정확" 수준으로 100% 굳힌 뒤 착수.** 지금은 앱 내 "찾는 지원금 종류" 한 문항으로 인접 수요를 데이터로 확인 → 확인된 **단 1개 버티컬만** 추가.

### 3개월
9. **격주/월간 동네 다이제스트** 정기 발송 시작, 구독 구조 안착 [R3]
10. **조리원 1곳 수동 컨시어지** 3개월 검증, 예약 1~2건 실적 확보 [M3]
11. **SEO 파일럿 3~5개 구** 고품질 페이지 + 웹 광고, 순위/노출 실측 [축2]
12. **"동네 지원금 한 장" PDF** 자동 생성해 부산 조리원·주민센터 비치 [축3]

## 6. 🚫 하지 말 것 / 리스크

### 하지 말 것 (kill + revise의 "버릴 것")
- **스폰서 슬롯 정액판매 / B2B 데이터 대시보드 납품** — 월 164명·구당 노출 수십 회에 월 5~15만원 낼 업체 없음(닭-달걀). SaaS 판매주기 수개월~수년·법인 요건 → 현 단계 완전 시기상조. **폐기.**
- **검증 데이터셋 B2B 구독 판매** — 지원금은 공개 데이터라 지불의사 원천적으로 낮음, 부산만 커버는 상품성 없음, 오류 시 법적 책임이 1인에게, SLA가 1인 운영과 충돌. **폐기.** (남긴다면 "유료 판매" 아닌 "무료 제휴로 노출과 물물교환")
- **체크리스트 3단계 게임화** — 신청이 앱 밖에서 끝나 돌아올 동기 없음. **폐기**(카피만 흡수)
- **주간 다이제스트** — 데이터 변동 속도와 충돌, 억지 발송이 알림 차단 유발. 격주/월간으로 대체
- **전국 226개 커버 / 대출 직접 수익화 / 리워드 유입 이벤트 반복** — 각각 신뢰 자멸·규제 위반·지표 오염

### 상시 리스크
- **틀린 마감일/미수령 총액 = 신뢰 즉사.** 데이터 오류가 "정확·최신"이라는 유일한 자산을 훼손. 기준일·출처 항상 표기, LLM 추출값은 사람 검증 후에만 발송
- **토스 플랫폼 의존**: 예약 푸시·발송 한도·리워드 강제 시청·딥링크 제약 모두 미검증 → 각 기능 착수 전 정책 확인 선행
- **지표 착시 경계**: eCPM 86,000원은 노출 28회 극소표본(신뢰구간 넓음), "40대 최다"는 6/30 이벤트 왜곡 가능성 → 코호트 분리 측정 전엔 이 신호 위에 전략 세우지 말 것
- **리텐션을 성공지표로 오해 금지**: 유틸리티는 one-and-done이 정상. 판단지표는 D1 리텐션이 아니라 **"재방문 시 광고 노출 완료율 / 마감 알림 반응률(CTR 1.9% 기준)"**
